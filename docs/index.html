Azure DevOps Agent
on Docker

Self-hosted agent on Alpine Linux. Multi-architecture, lightweight and with automatic releases.

Quick start Docker Hub →
$ docker pull neytor/azure-devops-agent:latest
Features

Everything you need

🏔️

Alpine Linux

Ultra-lightweight base image (~5 MB). Smaller attack surface, fast startup.

🔄

Multi-architecture

Native support for amd64 and arm64. Runs on servers, SBCs and Apple Silicon without changes.

📦

Auto-release

Detects new upstream agent versions and publishes automatically with changelog.

🐳

Docker-in-Docker

Docker socket mounted from the host. Your pipelines can build and push images.

⚡

Scalable

Spin up multiple agents with a single command using --scale.

🔒

Secure secrets

Environment variables via .env. Nothing hardcoded in the image.

Quick start

Up and running in 3 steps

1

Create the environment file

You need your organization URL and a PAT with Agent Pools permissions.

.env
AZP_URL=https://dev.azure.com/YOUR_ORG
AZP_TOKEN=YOUR_PAT
AZP_POOL=Default
2

Create the compose.yaml

compose.yaml
name: work_agent

services:
  azure-agent:
    image: neytor/azure-devops-agent:latest
    restart: unless-stopped
    env_file:
      - .env
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
3

Start the agent

terminal
# Single agent
$ docker compose up -d

# Three agents in parallel
$ docker compose up -d --scale azure-agent=3

# View logs
$ docker compose logs -f

# Stop
$ docker compose down
